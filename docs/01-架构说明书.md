# WFM 中央楼层监控系统 — 架构说明书

> 版本：1.0.1　｜　适用：`com.ibm.waferview.wfm:wfm-componet-client`
> 本文描述系统整体架构，重点说明在既有 2D SVG 厂图监控之上新增的 **3D 数字孪生厂图** 子系统。

---

## 1. 系统定位

FAB（晶圆厂）中央楼层监控系统（WFM / WAFER VIEW MES 客户端组件），用于产线管理：实时展示每层楼的**设备 Layout、设备/Port 状态、AMHS（OHT）小车**。

提供两套并存、可切换的可视化视图：

| 视图 | 形态 | 入口 |
|------|------|------|
| **2D 厂图**（既有） | SVG + D3，按元素 id 着色 | `/service/api/wfm_monitor` |
| **3D 数字孪生**（新增） | three.js / WebGL，footprint 拉伸 + glTF 模型 | `/service/api/wfm_monitor3d` |

两者**共用同一套后端数据接口**，颜色语义一致。

## 2. 总体架构

```
┌──────────────────────────── 浏览器（前端） ────────────────────────────┐
│  2D 监控 monitor.jsp           3D 监控 monitor3d.jsp                     │
│  jQuery + D3 + SVG             jQuery + three.js(r128)                   │
│  按 SVG 元素 id 着色            OrbitControls / GLTFLoader / CSS2D        │
│        │  ┌──────────── 切换按钮（独立页面互跳） ────────────┐  │        │
│        └──┤                                                  ├──┘        │
└───────────┼──────────────────── HTTP（/service/*）────────────┼─────────┘
            │  getSvgFile · getEqpStatusMaps · getOHTStatus       │
            │  getColorMap · listFloorDef · getEqpModelDefs        │
┌───────────▼─────────────────────────────────────────────────────────────┐
│                    Spring Boot 2.4.1（嵌入式 Tomcat:6080）                 │
│  DispatcherServlet 映射 /service/*  ·  context-path /wfm-component-client │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌──────────────────┐   │
│  │ Controller  │→│  Service    │→│   DAO        │→│  MyBatis Mapper   │   │
│  │ (REST/MVC)  │ │ (业务逻辑)   │ │ (接口)        │ │  XML (mapper/*)   │   │
│  └─────────────┘ └─────────────┘ └─────────────┘ └──────────────────┘   │
│  Spring Security + JWT  ·  Druid 连接池  ·  Log4j2  ·  i18n               │
│  WfmCronJob（@Scheduled）：定期把 DB 状态写入 JSON 缓存                     │
└───────────┬───────────────────────────────────────┬──────────────────────┘
            │ JDBC(DB2 Driver)                        │ 文件读写
   ┌────────▼─────────┐                     ┌─────────▼──────────────────────┐
   │   IBM DB2         │                     │  wfm_file/                      │
   │  schema = WFM     │                     │   data/ file.stat.json (设备状态)│
   │  设备/状态/WIP/    │                     │        file.color.json (颜色)    │
   │  报警/楼层/用户     │                     │        file.mcp.json (OHT)       │
   └──────────────────┘                     │        model.registry.json (3D 模型)│
                                             │   svg/<F1|F2|…>/FabView.jsp (厂图)│
                                             └─────────────────────────────────┘
```

**关键机制——读写分离的状态缓存**：实时查询不直接打 DB，而由后台定时任务 `WfmCronJob` 周期性地把 DB 状态序列化到 `wfm_file/data/*.json`；前端轮询的 `getEqpStatusMaps`/`getOHTStatus` 优先读这些 JSON 文件（读不到再回退查 DB）。这降低了大屏高频轮询对 DB 的压力。

## 6. 3D 数字孪生子系统架构（重点）

### 6.1 设计原则
1. **几何来源仍是 SVG**——复用既有 `wfm_file/svg/<floor>/FabView.jsp`，不另建几何数据；2D/3D 共享同一份厂图，编辑/上传方式不变。
2. **数据接口零改动**——3D 复用 `getEqpStatusMaps`/`getColorMap`/`listFloorDef`，着色字段与 2D 完全一致。
3. **Port 为一等公民**——端口在 3D 中醒目、可点、按 `portStateColor` 着色（产线管理核心看点）。
4. **混合几何、渐进式数字孪生**——默认 2D footprint 拉伸成盒体；当某设备登记了真实 glTF 模型则加载模型，无模型自动回退拉伸。

### 6.2 渲染管线（`monitor3d.js`）
```
listFloorDef ─► 楼层下拉
getEqpModelDefs ─► 模型注册表(g_modelDefs)
        │
切层 ─► getSvgFile(floorNo) ─► 注入隐藏 DOM(#svgGeomHost)
        │
getEqpStatusMaps ─► buildFloor():
   每台设备(其 id 存在于 SVG)：
     getBBox()+getCTM() 还原根坐标矩形
     ├─ 注册表有 glTF ─► addEquipmentGltf()：按 footprint 缩放/落地/居中；
     │                    端口状态绑定到命名节点(portNodeMap)，无节点则合成端口块
     └─ 否则           ─► addEquipment() 拉伸盒体(顶面 curStateIdColor / 边框 eqpModeColor)
                          + addPort() 沿设备前缘合成端口块(portStateColor)
        │
每 5s 轮询 getEqpStatusMaps ─► applyStatus()：仅按 id 查表改材质颜色，不重建几何
```

### 6.3 关键技术点
- **坐标还原**：用 `element.getBBox()`（footprint）+ `element.getCTM()`（综合元素自身 transform）映射到 SVG 根坐标系，再归一化居中到 3D 地平面。各层 SVG 坐标系/缩放不同，按层动态读取根范围。
- **glTF 端口绑定**：注册表 `portNodeMap` 把 `portId` 映射到模型节点名；加载后按名查找节点、克隆材质后重着色（避免污染共享材质）。
- **健壮性**：`g_buildToken` 丢弃切层后过期的异步 glTF 回调；模型加载失败自动回退拉伸；SVG 缺端口元素时合成端口（FAB load port 排布）。
- **交互**：OrbitControls 旋转/缩放/平移；Raycaster 点击设备复用既有详情弹窗 `wfmWfviewEqpt`；CSS2D 标签 + 悬停 tooltip。

### 6.4 单层范围
3D 当前为**单层显示**（不做多楼栋堆叠），仅渲染**设备 + Port**。OHT（小车/轨道）为后续阶段，依赖 SVG 中的 `VHC_/PORT_` 轨道站点元素与 MCP 车辆数据。

## 7. 安全架构

- 认证：JWT Token + Session 双机制，BCrypt 加密，支持 Remember-Me。
- 授权：`SecurityConfig` 白名单（`permitAll`）放行登录、监控页与只读数据接口、静态资源（`*.js/*.css/*.svg/*.glb/*.gltf/*.bin` 等）。
- 3D 新增放行：`/service/api/wfm_monitor3d`、`/service/api/getEqpModelDefs`、`*.glb/*.gltf/*.bin`。
- 注：当前未启用 `anyRequest().authenticated()`，未匹配白名单的请求由 JWT 过滤器与默认策略处理。

## 8. 关键设计决策与取舍

| 决策 | 取舍 |
|------|------|
| 3D 复用 SVG 几何而非新建模型库 | ✅ 单一几何来源、2D/3D 同步、改动最小　❌ 受 SVG 精度限制（用拉伸近似真实外形，故引入 glTF 升级路径） |
| glTF 混合 + 拉伸回退 | ✅ 无模型也能跑、可逐台升级为数字孪生　❌ 需维护模型注册表与节点命名约定 |
| 状态走 JSON 文件缓存（定时任务生成） | ✅ 抗高频轮询、降低 DB 压力　❌ 有秒级延迟（按 cron 周期） |
| 独立 3D 页面而非内嵌切换 | ✅ 与 2D 解耦、互不影响、可单独演进　❌ 两页有少量骨架重复 |
| three.js global build + `<script>` 引入 | ✅ 与既有 JSP 资源加载方式一致、无需打包工具　❌ 非 ES Module，全局命名空间 |
