# WFM 中央楼层监控系统 — 运维文档

> 面向部署与运维：环境、构建、启停、配置、访问、数据文件、3D 资源维护、日志与故障排查。

---

## 1. 环境要求

| 项 | 要求 |
|----|------|
| JDK | 1.8+ |
| Maven | 3.6+（首次 `spring-boot:run` 需联网拉取插件） |
| 数据库 | IBM DB2，schema `WFM`（需可达，如 dev：`10.196.61.10:50001/CZMMDBD`） |
| 端口 | 6080 可用 |
| 浏览器 | 支持 WebGL（3D 视图依赖；大屏机需确认显卡/驱动可用） |
| 目录权限 | `wfm_file/` 可读写（运行时读写 JSON 缓存与读取 SVG） |

---

## 2. 构建与打包

```bash
mvn clean package
# 产出：target/wfm-componet-client-1.0.1.jar（fat jar，已内含 DB2 驱动）
```

---

## 3. 启动 / 停止

### 开发运行
```bash
mvn spring-boot:run
# 启动成功日志：Tomcat started on port(s): 6080 (http) with context path '/wfm-component-client'
#               Started WfmSpringBootApplication in N seconds
```
> 不要加 `-o`（离线）：会因缺少 `spring-boot-maven-plugin` 依赖而失败。

### 生产运行
```bash
java -jar target/wfm-componet-client-1.0.1.jar
# 指定环境：--spring.profiles.active=prod
# 工作目录需能看到 wfm_file/（FileUtil 以 user.dir 为基准定位）
```

### 停止
```bash
pkill -f spring-boot:run        # 或 kill <pid>
lsof -nP -iTCP:6080 -sTCP:LISTEN   # 确认端口已释放
```

## 4. 配置与多环境

- 切换环境：改 `application.yml` 的 `spring.profiles.active`（dev/tst/prod），或启动参数 `--spring.profiles.active=prod`。
- 换库：改对应 `application-<env>.yml` 的 `spring.datasource.{url,username,password}`，并确保 `lib/db2jcc4.jar` 存在。
- 定时任务频率：`eqpstatusupdate.schedules`（设备状态，默认每分钟）、`updateVehicleData.schedules`（OHT，默认每 5 秒）。

## 5. 访问地址（重要）

⚠️ **裸上下文根 `/wfm-component-client/` 会 404**（无处理器映射），属正常。请使用 `/service/` 下的地址：

| 用途 | URL |
|------|-----|
| 登录页 | `http://<host>:6080/wfm-component-client/service/auth/preLogin` |
| 2D 监控 | `http://<host>:6080/wfm-component-client/service/api/wfm_monitor` |
| **3D 监控** | `http://<host>:6080/wfm-component-client/service/api/wfm_monitor3d` |

进入 3D 后用左上角下拉框切换楼层；F2 数据最全（设备/端口多）。2D/3D 页面顶部有互相切换按钮。

## 6. 外部依赖

### DB2
- 必须可达且 schema `WFM` 有相应视图/表（设备、状态、WIP、报警、楼层、用户、颜色等）。
- 连接由 Druid 管理，可在日志看到 MyBatis SQL（dev 级 DEBUG）。

### wfm_file 目录
| 路径 | 用途 | 谁写 | 谁读 |
|------|------|------|------|
| `data/file.stat.json` | 设备+端口状态缓存 | `WfmCronJob.execute()`（每分钟） | `getEqpStatusMaps` |
| `data/file.mcp.json` | OHT/车辆缓存 | `WfmCronJob.updateVehicleData()`（每 5 秒） | `getOHTStatus` |
| `data/file.color.json` | 颜色定义 | 维护/导出 | `getColorMap` |
| `data/model.registry.json` | **3D 设备模型注册表** | 人工维护 | `getEqpModelDefs` |
| `svg/<F1\|F2\|…>/FabView.jsp` | 各层厂图 SVG | 上传（厂图管理） | `getSvgFile`（2D/3D 共用） |

> 部署务必确保 `wfm_file/` 存在且可读写，否则状态缓存无法生成、厂图无法加载。

## 7. 3D 功能运维

### 7.1 新增/更新楼层厂图
- 把该层 SVG 放到 `wfm_file/svg/<floorId>/FabView.jsp`，设备元素 `id` 必须等于设备号（与 DB 设备 id 一致）；端口元素 `id` 等于 `mcpPortID`。
- 2D 与 3D 自动共用，无需改代码。

### 7.2 接入真实 glTF 模型
1. 模型文件放 `src/main/webapp/models/`（部署后对应静态目录）；`*.glb/*.gltf/*.bin` 已在安全白名单。
2. 编辑 `wfm_file/data/model.registry.json` 增加设备→模型映射（含 `portNodeMap`，详见《技术文档》§7）。
3. 刷新 3D 页生效（注册表实时读取，无需重启）。
4. 无模型的设备自动回退为 footprint 拉伸盒体。

### 7.3 端口看不到颜色？
端口按 `portStateColor` 着色；若实时库端口为默认 `#fff`（产线空闲），端口呈白色，属正常。设备主体仍按 E10 状态着色。

## 9. 常见故障排查

| 现象 | 原因 | 处理 |
|------|------|------|
| 访问 `/wfm-component-client/` 返回 **404** | 裸上下文根无映射 | 改用 `/service/...` 地址（§5） |
| 启动报 `ClassNotFoundException: com.ibm.db2.jcc.DB2Driver` | DB2 驱动不在运行时类路径 | 确认 `pom.xml` 有 db2jcc4 的 system 依赖、`lib/db2jcc4.jar` 存在（《技术文档》§2.2） |
| `spring-boot:run` 报插件离线无法解析 | 用了 `-o` 离线模式 | 去掉 `-o`，联网运行 |
| 启动卡住/失败在数据源 | DB2 不可达或账号错 | 检查 `application-<env>.yml` 的 url/账号、网络连通性 |
| 端口 6080 被占用 | 已有进程 | `lsof -nP -iTCP:6080 -sTCP:LISTEN` 找到并 kill |
| 3D 页空白/报错 | 浏览器不支持 WebGL，或 three.js 资源 404 | 确认显卡/浏览器；确认 `/service/three/*.js` 可访问 |
| 3D 某层设备很少 | 实时库设备 id 与该层 SVG 重叠少 | 正常（数据现状）；检查 SVG 设备 id 与 DB 是否一致 |
| 3D 看不到端口 | 该层 SVG 无端口元素 / 端口状态为空闲白色 | 合成端口仍会显示；颜色随真实 `portStateColor` 变化 |
| glTF 模型不加载 | 文件路径错 / 未在安全白名单 / 注册表 url 错 | 确认 `models/` 下文件、`*.glb/*.gltf/*.bin` 放行、`model.registry.json` 的 `url` |
| 改了 `monitor3d.js` 不生效 | webapp 资源在启动时已复制到 target | 重启 `spring-boot:run` 或重新打包 |

## 10. 备份与回滚

- **状态缓存**：`file.stat.json`/`file.mcp.json` 由定时任务每分钟/每 5 秒覆盖重写，无需备份；如需保留某快照可另存（如 `file.stat.json.bak`）。
- **配置/厂图/模型注册表**：`application-*.yml`、`wfm_file/svg/*`、`model.registry.json` 建议纳入版本管理或单独备份。
- **代码回滚**：Git 仓库 `pillarcao/MyWFM3D`，按 commit 回滚（`git revert` / `git checkout <commit>`）。
