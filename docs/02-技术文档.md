## 2. 构建与依赖

### 2.1 构建
```bash
mvn clean package          # 产出 target/wfm-componet-client-1.0.1.jar（含 BOOT-INF/lib 的 DB2 驱动）
mvn spring-boot:run        # 开发运行（需联网首次拉取 spring-boot-maven-plugin）
```

### 2.2 DB2 驱动（关键）
`lib/db2jcc4.jar` 通过两种方式纳入：
- **编译**：`maven-compiler-plugin` 的 `<extdirs>${project.basedir}/lib</extdirs>`。
- **运行**：声明为 **system 范围依赖**，使 `spring-boot:run` 运行时类路径包含驱动：
  ```xml
  <dependency>
    <groupId>com.ibm.db2</groupId><artifactId>db2jcc4</artifactId><version>4.0</version>
    <scope>system</scope><systemPath>${project.basedir}/lib/db2jcc4.jar</systemPath>
  </dependency>
  ```
- **打包**：`spring-boot-maven-plugin` 配 `includeSystemScope=true`，且 `lib/*.jar` 作为资源复制到 `BOOT-INF/lib`。
> 若缺此 system 依赖，`mvn spring-boot:run` 会报 `ClassNotFoundException: com.ibm.db2.jcc.DB2Driver`。

### 2.3 three.js（3D 前端库）
本地放置于 `src/main/webapp/three/`，在 `monitor3d.jsp` 中以 `<script src="<%=baseUrl%>three/xxx.js">` 引入（与 `d3/d3.js` 同模式）。版本 r128 global build，全局命名空间 `THREE.*`、`THREE.OrbitControls`、`THREE.GLTFLoader`、`THREE.CSS2DRenderer`。

## 5. 数据模型（前端着色相关）

`getEqpStatusMaps` → `eqpStatusList[]`：
```jsonc
{
  "eqpStatus": {
    "eqpId":"DZ01", "eqpName":"...", "eqpCategory":"U", "eqpMode":"MANU",
    "e10State":"READY", "curStateId":"READY",
    "curStateIdColor":"#0033FF",   // ← 设备主体/盒体顶面颜色
    "eqpModeColor":"#990000",      // ← 盒体边框颜色
    "stateChgTime4Text":"8D19H", "fontColor":"#010101"
  },
  "eqpPortStatusList":[            // ← Port（3D 核心看点）
    { "eqpId":"DZ01","portId":"01","mcpPortID":"DZ01_01",
      "portState":"...", "portStateColor":"#33FF00" }   // ← 端口颜色
  ],
  "eqpChamberStatusList":[ ... ],  // 3D 当前未渲染
  "inprocessLotIds":[...], "wipCnt":0
}
```
- `colorMap`：`COLOR_EQP_STATE_BG/FG`、`COLOR_EQP_MODE` 等状态→颜色字典（2D header 图例用）。
- `getOHTStatus` → `mcpInfo{ mcpName, fillColor, vehicleList[currentLocation,vehicleName,emptyFlag,fillColor], stationList[stationID,carID,color] }`。
- 几何对应关系：SVG 中 `<path id=eqpId>` = 设备；`id=mcpPortID` 的元素 = 端口；`VHC_xxxx` = OHT 轨道段；`PORT_<stationID>` = 站点。

> 注：实时库中端口可能为默认色 `#fff`（空闲），属正常；有载具/状态时按 `portStateColor` 着色。

## 6. 3D 前端实现（`monitor3d.js` 关键函数）

| 函数 | 职责 |
|------|------|
| `init3D()` | 建 Scene/Camera/WebGLRenderer/CSS2DRenderer/灯光/地面/OrbitControls/GLTFLoader/Raycaster，启动 `animate()` |
| `loadModelDefs(cb)` | 拉模型注册表 → `g_modelDefs` |
| `loadFloor()` | `getSvgFile` 注入 `#svgGeomHost` → `fetchStatusAndBuild(true)` |
| `fetchStatusAndBuild(rebuild)` | 拉 `getEqpStatusMaps`；首次/切层 `buildFloor`，否则 `applyStatus`；启动 5s 轮询 |
| `rectFromElement(el)` | `getBBox()`+`getCTM()` → 根坐标轴对齐矩形 |
| `toWorld(x,y)` | SVG 坐标 → 3D (x,z)（居中 + 归一化 `g_scale`） |
| `buildFloor(list)` | 遍历设备：`resolveModelEntry` 分流 glTF / 拉伸 |
| `addEquipment()` / `addPort()` | 拉伸盒体（顶面 `curStateIdColor`、边框 `eqpModeColor`）/ 端口块（`portStateColor`，无 SVG 元素时合成于设备前缘） |
| `addEquipmentGltf()` | 加载 glTF、按 footprint 缩放落地、`portNodeMap` 绑定端口节点、缺节点回退合成 |
| `recolorObject(obj,color)` | 克隆材质后重着色（color + emissive） |
| `applyStatus(list)` | 仅按 id 查 `meshByEqpId`/`meshByPortId` 改色，不重建 |
| `onPointerMove/onClick` | Raycaster 悬停 tooltip / 点击设备开 `wfmWfviewEqpt` |

可调参数（文件顶部）：`TARGET_WIDTH`（整层归一化宽度）、`EQP_HEIGHT`（类别→代理高度）、`PORT_RAISE`/`PORT_HEIGHT`（端口块）。

## 7. 真实 glTF 模型接入

1. 把 `.glb` / `.gltf`（含 `.bin`）放到 `src/main/webapp/models/`（或部署后的对应静态目录）。
2. 在 `wfm_file/data/model.registry.json` 增加条目：
   ```jsonc
   {
     "<eqpId>": {
       "source": "gltf",
       "url": "models/<file>.glb",          // 相对 baseUrl(/service/)
       "transform": { "scaleToFootprint": true, "rotateY": 0, "offsetY": 0 },
       "portNodeMap": { "<portId>": "<模型中的端口节点名>" }
     }
   }
   ```
   - 也支持**按类别**绑定：键用 `"CAT:<eqpCategory>"`。
   - `scaleToFootprint:true` 时模型 XZ 自动贴合设备 SVG 矩形；可叠加 `scale` 微调。
3. **建模约定**：模型中的端口需有可识别节点名（如 `Port_1`），`portNodeMap` 才能把端口状态色绑上；无对应节点的端口自动回退为合成端口块。
4. 刷新 3D 页即生效（注册表在请求时实时读取，无需重启）。

> 演示资产：`models/demo_eqp.gltf`（立方体 + `Body/Port_1/Port_2` 节点），注册表已将其绑定到 `DZ01`(F1) 与 `BS21`(F2) 作为混合几何示范。

## 8. 定时任务（`watchdog/WfmCronJob`，`@EnableScheduling`）

| 方法 | cron | 行为 |
|------|------|------|
| `execute()` | `eqpstatusupdate.schedules`（每分钟） | 查询全部设备状态 → 写 `file.stat.json` |
| `updateVehicleData()` | `updateVehicleData.schedules`（每 5 秒） | 查询 MCP/车辆 → 写 `file.mcp.json` |

前端轮询读取这些 JSON，实现“准实时”而不直连 DB。

## 9. 约定与注意

- **URL 前缀**：所有接口/页面在 `/service/*` 下；前端用相对路径 `api/...`（页面 `<base href>` = `.../service/`）。
- **国际化**：`?lang=zh_CN|zh_TW|en_US` 或 Session `wfm-language`。
- **JS 资源更新**：`webapp` 资源在 `spring-boot:run` 启动时复制到 `target/classes/META-INF/resources`；改 `monitor3d.js` 等需重启/重打包才生效。`model.registry.json` 在 `wfm_file/` 外部目录，请求时实时读取，无需重启。
